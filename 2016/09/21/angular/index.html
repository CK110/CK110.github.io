2.ViewContainerRef:

Represents a container where one or more Views can be attached.

  • 代表一个容器,可以附加一个或多个视图。

The container can contain two kinds of Views. Host Views, created by instantiating a Component via createComponent, and Embedded Views, created by instantiating an Embedded Template via createEmbeddedView.

  • 容器可以包含两种视图。宿主视图创建的实例化一个组件通过createComponent和嵌入视图,通过createEmbeddedView由嵌入式模板实例化。

The location of the View Container within the containing View is specified by the Anchor element. Each View Container can have only one Anchor Element and each Anchor Element can only have a single View Container.

  • 视图容器内包含视图的位置是由锚指定的元素。每个视图容器只能有一个锚元素和每个锚元素只能有单一视图容器。

Root elements of Views attached to this container become siblings of the Anchor Element in the Rendered View.

  • 根元素的视图附加到这个容器成为兄弟姐妹的锚元素呈现视图。

To access a ViewContainerRef of an Element, you can either place a Directive injected with ViewContainerRef on the Element, or you obtain it via a ViewChild query.

  • 访问ViewContainerRef的一个元素,您可以将指令注入ViewContainerRef元素,或者你得到它通过ViewChild查询。

3.ChangeDetectionStrategy

Class Overview

class ChangeDetectionStrategy {

    OnPush 

    Default 

}

Class Description

Describes within the change detector which strategy will be used the next time change detection is triggered.

  • 描述了在改变探测器将使用哪种策略触发下一次变化检测。

Class Export

export enum ChangeDetectionStrategy

Class Details

OnPush

OnPush means that the change detector’s mode will be set to CheckOnce during hydration.

  • OnPush意味着改变探测器的模式将被设置为CheckOnce

Default

Default means that the change detector’s mode will be set to CheckAlways during hydration.

  • 意味着改变探测器的模式将被设置CheckAlways

##4.OnInit
Implement this interface to execute custom initialization logic after your directive’s data-bound properties have been initialized.

  • 实现这个接口来执行自定义的初始化逻辑指令后的数据绑定属性被初始化。

ngOnInit is called right after the directive’s data-bound properties have been checked for the first time, and before any of its children have been checked. It is invoked only once when the directive is instantiated.

  • ngOnInit 被称为第一次指令的数据绑定属性检查之后,在这之前的children已经检查。它只调用一次指令时实例化。

##5.ngAfterViewInit
Implement this interface to get notified when your component’s view has been fully initialized.

  • 实现这个接口来得到通知当你已经完全初始化组件的视图。
